# 安全说明

## 环境变量管理

本项目所有敏感配置（数据库连接串、API 密钥、签名密钥等）均通过**环境变量**注入，代码中仅通过 `process.env.XXX` 访问。

### 本地开发

1. 复制 `.env.example` 为 `.env`
2. 将 `< >` 内的占位符替换为真实值
3. **绝对不要**将 `.env` 文件提交到版本控制

### 生产部署

- 环境变量由部署平台（Railway）的环境变量管理功能配置
- 不使用 `.env` 文件
- 通过平台 UI 或 CLI 设置变量

## 安全检查清单

- [x] `.env` 已在 `.gitignore` 中排除
- [x] 代码中无硬编码的密钥、连接串或令牌
- [x] `demo/` 文件夹中的示例代码未被生产代码引用
- [x] 所有敏感信息仅通过 `process.env.XXX` 访问

## 报告安全问题

如发现安全漏洞，请通过项目维护者联系方式私下报告，不要在公开 Issue 中披露。
